Arquitetura

Como funciona

O pipeline une dois datasets públicos, aplica Transfer Learning com MobileNetV2 e converte o resultado para rodar no navegador via TensorFlow.js.

01

Super Dataset

Mesclagem de TrashNet e Garbage Real World (Kaggle) em 6 categorias. Imagens com fundos reais — mãos, rua, grama — aumentam a robustez.

→
02

Transfer Learning (MobileNetV2)

Base pré-treinada no ImageNet (congelada) + camadas Dense próprias. Apenas 10 épocas no Google Colab GPU — mais rápido e mais preciso que treinar do zero.

→
03

Conversão TF.js

Modelo salvo como .h5 e convertido com tensorflowjs_converter. Roda 100% no browser via WebGL — zero backend, zero servidor.

Detalhe da Arquitetura

Input 224 × 224 × 3 px imagem bruta RGB
↓
Rescaling ÷ 127.5 − 1 normaliza para [−1, 1] (padrão MobileNet)
↓
MobileNetV2 Base pré-treinada (ImageNet) — congelada ~2,2M parâmetros não treináveis
↓
GlobalAvgPool GlobalAveragePooling2D achata o feature map
↓
Dropout rate = 0.2 regularização
↓
Dense 6 neurônios · softmax saída: probabilidade de cada classe

Métricas do Modelo

MobileNetV2
Arquitetura Base

Transfer Learning a partir de pesos ImageNet. Base congelada durante o treino.

2 fontes
Super Dataset

TrashNet + Garbage Real World (Kaggle), mesclados em 6 classes com shuffle e split 80/20.

10
Épocas de Treino

Menos épocas que uma CNN do zero — Transfer Learning converge mais rápido e com menor risco de overfitting.

~43 MB
Tamanho do Modelo TF.js

Distribuído em shards para carregamento progressivo no browser via TF.js 4.20.

Configuração de Treinamento

Batch Size32
Input Shape224 × 224 × 3
OptimizerAdam (lr=0.001)
LossSparse Categorical Crossentropy
AugmentationFlip H+V · Rotate 20% · Zoom 20%
AmbienteGoogle Colab (GPU)
